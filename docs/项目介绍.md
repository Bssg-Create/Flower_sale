# 花卉销售管理系统 - 项目介绍文档

## 1. 项目概述

### 1.1 项目背景
随着人们生活水平的提高，花卉消费市场日益繁荣。传统的花卉销售模式已无法满足消费者多样化的需求，特别是年轻消费者对个性化、定制化产品的追求。本系统旨在打造一个集花卉展示、销售、定制为一体的综合性平台。

### 1.2 项目目标
- 提供便捷的花卉浏览和购买体验
- 实现花卉DIY定制功能，满足个性化需求
- 提供完善的后台管理功能，便于运营维护

### 1.3 创新点
**DIY花束定制模块**：用户可以自主选择花材、包装方式，通过拖拽方式进行花束设计，实时预览效果。

#### 4.1.3 DIY花束定制（核心功能）
```
流程说明：
1. 用户进入DIY页面
2. 选择包装样式（圆形、心形、长形等）
3. 从左侧花材列表拖拽花朵到包装区域
4. 调整花朵位置和数量
5. 实时计算价格
6. 保存或加入购物车
```

## 7. 接口设计

### 7.1 用户认证接口

| 接口 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| 登录 | POST | /api/auth/login | 用户登录 |
| 注册 | POST | /api/auth/register | 用户注册 |
| 退出 | POST | /api/auth/logout | 用户退出 |

### 7.2 花卉管理接口

| 接口 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| 花卉列表 | GET | /api/flower/list | 获取花卉列表 |
| 花卉详情 | GET | /api/flower/{id} | 获取花卉详情 |
| 添加花卉 | POST | /api/flower | 添加花卉 |
| 更新花卉 | PUT | /api/flower/{id} | 更新花卉 |
| 删除花卉 | DELETE | /api/flower/{id} | 删除花卉 |

### 7.3 DIY模块接口

| 接口 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| 花材列表 | GET | /api/diy/flowers | 获取DIY可用花材 |
| 包装类型 | GET | /api/diy/packages | 获取包装类型 |
| 保存花束 | POST | /api/diy/save | 保存DIY花束 |
| 花束列表 | GET | /api/diy/list | 获取用户花束列表 |
| 删除花束 | DELETE | /api/diy/{id} | 删除花束 |

- JDK 17+
- Maven 3.8+

### 8.2 启动步骤

**后端启动：**
```bash
cd flower-web
mvn clean spring-boot:run
```

```bash

npm install

```

## 10. 总结

本花卉销售管理系统旨在提供一个现代化的花卉电商平台，核心创新点在于DIY花束定制功能，允许用户通过拖拽方式自主设计花束。系统采用Spring Boot + Vue的前后端分离架构，具有良好的扩展性和可维护性。
